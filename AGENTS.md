# AGENTS

This repo is a React Native (Expo) app that controls an ESP32-based quadruped robot over local WiFi (WebSocket).

## Project Snapshot

- App entry: App.tsx
- UI + logic: src/ (components, screens, services, store, theme)
- Native Android: android/ (generated by Expo prebuild; avoid changes unless necessary)
- Build script: build-apk.js

## App Architecture (Quick Map)

- State: src/store/useRobotStore.ts (Zustand)
- Network: src/services/websocket.ts (ESP32 WebSocket)
- Navigation: src/navigation/AppNavigator.tsx (bottom tabs)
- Screens: src/screens/ (Connect, Control, Calibration, Head, Settings)
- UI parts: src/components/ (DPad, GaitButton, ServoSlider, StatusBar, SectionCard)
- Theme: src/theme/colors.ts + typography.ts

## Key Commands

- Install: npm install
- Run dev: npm start

### APK Build

Default (small, arm64 only):

node build-apk.js

No-bump (keep version and versionCode):

node build-apk.js --no-bump

Universal APK (all ABIs):

node build-apk.js --universal

Other variants:

node build-apk.js --abi armeabi-v7a
node build-apk.js --abis arm64-v8a,x86_64

Build tips:

- Use --no-bump for repeat builds
- Use --clean if Gradle cache gets flaky
- Use --force-prebuild if android/ is out of sync

### Output Naming

APKs are written to releases/ with:

- QUADRUPED-App-vX.Y.Z-arm64-v8a.apk
- QUADRUPED-App-vX.Y.Z-armeabi-v7a.apk
- QUADRUPED-App-vX.Y.Z-arm64-v8a-x86_64.apk
- QUADRUPED-App-vX.Y.Z-universal.apk

If a name already exists, the build script appends a numeric suffix. Delete old APKs in releases/ to keep clean names.

## Versioning

- Source of truth: package.json version + app.json expo.version
- Android versionCode: app.json expo.android.versionCode
- build-apk.js bumps patch by default (use --no-bump to keep versions)

## Release Signing

Release builds are debug-signed unless a keystore is configured.
Set these (gradle.properties or env vars) to sign properly:

- RELEASE_STORE_FILE
- RELEASE_STORE_PASSWORD
- RELEASE_KEY_ALIAS
- RELEASE_KEY_PASSWORD

## Build Notes

- build-apk.js sets NODE_ENV=production if missing
- Prebuild is cached by a fingerprint; it is skipped when unchanged
- R8 minify/shrink is on by default for release builds

## Do / Don't

Do:

- Keep edits small and targeted
- Prefer changes in src/ over native android/ unless required
- Use build-apk.js for release builds
- Remove old APKs in releases/ to keep clean names

Don't:

- Edit generated android/ files unless unavoidable
- Bump versions when running quick local builds
- Commit releases/ APKs unless explicitly needed

## Troubleshooting

- R8 missing classes: check android/app/proguard-rules.pro and android/app/build/outputs/mapping/release/missing_rules.txt
- Prebuild issues: run node build-apk.js --force-prebuild
- Build cache weirdness: run node build-apk.js --clean
- Large APK size: use arm64-v8a or ABI-specific build
- Debug-signed release: configure RELEASE_* keystore values

## Adding a Screen or Component

Screen workflow:

- Create a new screen in src/screens/
- Wire it in src/navigation/AppNavigator.tsx
- Use theme tokens from src/theme/
- Keep touch targets >= 48dp

Component workflow:

- Add component in src/components/
- Reuse SectionCard, ServoSlider, or DPad patterns where possible
- Keep colors aligned to src/theme/colors.ts
